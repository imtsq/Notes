# 1. 考法1----精确计算

识别：题目没说约等于

## 1.1. 基础方法

（选项精度与算式一致） ----- 尾数法（直接看末尾或者最后两位）

- 选项最后一位都不同就看最后一位
- 有选项最后一位同就看两位

![](https://cdn.nlark.com/yuque/0/2025/png/32465899/1757407264263-afccd86c-2c1e-48fc-8b2a-223f48150ec9.png)

- 第一个末尾相加为2
- 第二个末尾是8，选不出，那么就是右边第二位8-3选A

对于下面这种![](https://cdn.nlark.com/yuque/0/2025/png/32465899/1757407441048-50ef6491-3fdb-4af0-8915-4ef1faccd5f1.png)认证了末位就是最后一位（这里主要是题和选项精确度一样，都是小数点后一位），那么就直接看最后一位

## 1.2. 当多个位数相加减

- 方法1：所有加法归类，所有减法归类，然后最后在计算，（当然也是对最后算一到两位）
- 方法2：看有没有直接能抵消的

# 2. 估算

- 题目有明确告诉你是估算

选项精度和算式不一致 ---- 高算低看（高位精算低位看）

什么是高算低看，

要结合选项

比如38929-27871

A 11.... B 10.... C 9....

这种首先看高位大概是11第三位也不需要借位，不需要结尾直接结束运算A

## 2.2. 多个数加减

逐高位运算

![](https://cdn.nlark.com/yuque/0/2025/png/32465899/1757409999228-74d137b8-939e-40f7-adff-05a4773c4ca7.png)

先 8 + 2 + 3 + 3 + 6 + 5 再后面第二位逐位

多位数相加减，首先是高位先算，然后高二位再算，如果同时结合答案

- 比如上面开始高位为27（这里不能直接算B因为后面可能近卫）
- 再算第二位7 + 3 + 8 + 2 + 2 + 2 = 24
- 270+ 24 = 294就不需要算后面的了（如果选项有29几 30几可能还要算因为后面进位问题）
