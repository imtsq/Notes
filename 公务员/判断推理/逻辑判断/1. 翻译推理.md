# 翻译规则
## 翻译规则前推后
典型：
- 如果……那么……
- ...........则.............
- .............就...........
- ............都............
- .............一定...........
- .......是.................的充分条件 （这个很少考）

注意： 这里是条件结论，不要用言语的做题，如:为了变美一定减肥：

可能觉得减肥导致美（错误思维），这里条件是<font style="color:#DF2A3F;">变美</font>-><font style="color:#DF2A3F;">减肥</font>了（条件结论）





1. 读文短，如果逻辑关系是（不要用言语方式做题，用数学思维）
    1. 如果......那么......
    2. .......则.....
    3. 只有.....就....（我们是这样有只要....就....因为后面又只有......才）
    4. .....都.....
    5. ......一定.....
    6. （其他能直接看出p->q的，大部分是上面几种）



总结： 如果，就，则，那么、都、一定（看到这些就是前推后，因为有些可以省略）

## 后推前
1. 只有……才……
2.  除非......否则不（否则和不一定连在一起）....
    1. 除非天地合，否则我不会与君绝（这句话有点拗口，你把<font style="color:#DF2A3F;">否则</font>翻译为<font style="color:#DF2A3F;">如果不这样</font>，否则不翻译为如果不这样不干嘛）
    2. 与君绝->天地合
3. 除非A 否则 B
    1. 除非交房租，否则搬出去
    2. 不搬出去->交房租
    3. 非B -> A
4. 不.....不......
    1. 名不正，则言不顺（言顺->名正）
    2. 其实一般加了则，则本来是前推后，这里不没了，就变成逆否

为什么上面的是后->前？

因为可能条件又多个

如：ABC同时满足才有D,   现在给个A是B的前提，只能说B完成的情况下A一定完成

## 特殊（并非前推后也并非后推前）
.....是.....的<font style="color:#DF2A3F;">基础/假设/前提/关键/必要/必不可少</font>

为什么不直接记忆后推前?

因为经常变化，可能交换语序

# 推理规则共性
## 逆否等价
1. A -> B
2. 非B -> 非A
3. 其他不满足，非A不能推
4. <!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/32465899/1779199077997-761a24b6-de31-4c10-bed4-8b890b982760.png)

## 递推规则
1. A -> B, B->C
2. A->B->C同时A->C

## 且与或
没什么要学到，写程序经常用

且：就是两个同时满足，学习好，就要认真学习并且有天赋

学习好->认真学 且天赋

或：就是二则满足其一，A或B

+ A
+ B
+ A+B
+ 想学习好，就要认真学或者天赋
+ 学习好->要不然认真学，要不然有天赋，要不然有天赋 + 认真学
    - 反之，如果学校好，没认真学就是有天赋

或知道一个不满足，能推出另一个满足

## 德摩根定律
就是且或互换

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/32465899/1779354274080-84c04e7c-945c-4fdf-9d58-7f781ee396bb.png)

推理：

1. A且B，是两个都正确才正确，否定是其中一个否就是否定，可能同时两个否定
2. A或B，其中一个正确就正确，否定，需要两个同时否
